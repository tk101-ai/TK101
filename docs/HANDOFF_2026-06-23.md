## 1. 환경 & 배포 (필독)
- **서버**: Tencent CVM 서울. self-hosted GitHub Actions runner.
- **배포 = main에 머지(또는 push)** → runner가 `docker compose up -d --build` + (백엔드 컨테이너 내) `alembic upgrade head`. **main 직접 push 금지, 항상 브랜치→PR→머지.** 머지가 곧 배포 트리거.
- **배포 확인**: `gh run list` / `gh run watch <id> --exit-status`.
- **컨테이너**:
  - `tk101-backend` — 운영(이미지 베이크). 코드는 `/app`.
  - `tk101-backend-dev` — `/srv`에 **워킹트리 마운트**(현재 체크아웃된 브랜치 코드 그대로 보임). DB=`tk101_dev`. import/마이그레이션/pytest 테스트용.
  - `tk101-postgres`(운영 DB), `tk101-qdrant`(검색), `tk101-frontend`(nginx).
- **포트**: 프론트 `localhost:8080`(nginx, `/api/`는 백엔드로 프록시, 타임아웃 300s). 백엔드 `127.0.0.1:8000`(직접 접근은 nginx 경유 권장).
- **DB 마이그레이션 head = 035** (운영 적용됨). 다음 마이그는 036, down_revision=035.
- **gh 인증됨**(tk101-ai).

## 2. 작업 방식 & 검증 게이트 (반드시 지킬 것)
흐름: **조사 → 구현 → 검증 → 브랜치 커밋 → PR → 머지(=배포) → 운영 검증 → 결과 보고**.
- **멈춰 묻는 건 단 둘**(CLAUDE.md §2): ① 되돌리기 어려운 운영 파괴(삭제/DROP/대량 마이그/영구삭제) ② 비용 큰 작업(대량 LLM·임베딩). 콘솔·외부계정·비밀번호·방화벽 콘솔 단계도 오너에게 넘김. 그 외 제품/UX/데이터모델은 합리적 안으로 자율 + 가정 명시.
- **검증 게이트(이걸 통과해야 "완료")**:
  - 백엔드: `python3 -m py_compile <편집파일>` + `docker exec tk101-backend-dev python -c "import app.main"`.
  - 프론트: **`cd frontend && npm run build`** (=`tsc -b && vite build`). ⚠️ `tsc --noEmit`는 `noUnusedLocals`를 못 잡아 미사용 import가 통과돼버림 → **반드시 `npm run build`로**.
  - DB 마이그: `docker exec tk101-backend-dev sh -c 'cd /srv && alembic upgrade head && alembic downgrade -1 && alembic upgrade head'` (적용/롤백/재적용) on `tk101_dev` 후 운영 배포.
  - 테스트: `docker exec tk101-backend-dev sh -c 'cd /srv && python -m pytest tests/ -q'` (현재 96개).
  - **배포 후 운영 검증 필수**: 라우트 등록은 `curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/...`(401=존재/200·422 등=존재, 404=미등록). 프론트는 빌드 산출 해시(`dist/assets/index-*.js`)와 `curl localhost:8080/` 응답의 해시 일치 확인. DB 컬럼은 asyncpg로 inspect.
- **커밋**: `<type>(<scope>): <subject>`, 끝에 `Co-Authored-By: ...`. `frontend/vite.config.ts`의 dev 프록시("커밋 안 함" 표식)는 항상 커밋 제외.
- **백로그**: 시작 시 `[ ]→[~]`, 완료 시 `[~]→[x]`(PR번호 포함). 완료분은 DONE 섹션으로 내린다.
- **동적 설계**: 계정/브랜드/부서/플랫폼 하드코딩 금지(DB/코퍼스 순회로 자동 반영).

## 3. 기술 스택 & 아키텍처 핵심
- **백엔드**: FastAPI + SQLAlchemy(async) + alembic, Python 3.12, `backend/pyproject.toml`(PEP621, dev에 pytest·black·ruff). 라우터 `app/routers/`, 서비스 `app/services/`, 모델 `app/models/`, 스키마 `app/schemas/`.
- **프론트**: React 19 + antd **v6** + recharts 2.x + vite + TS. 페이지 `frontend/src/pages/`, API 클라이언트 `frontend/src/api/`(axios 단일 인스턴스 `api/client.ts`, Bearer+401 인터셉터). 공통 에러: `utils/errorUtils.extractErrorDetail(e, fallback)`.
- **NAS 검색**(가장 복잡): Qdrant `docs_text`(Qwen3-Embedding-4B **2560d cosine**, 160만 청크) 단일소스. 적재는 외부 파이프라인 `/home/ubuntu/tk101-rag`(별도 repo). 검색 = **하이브리드**: 벡터 arm(의미) + 키워드 arm(정확, `text`/`path` 풀텍스트 MatchText 인덱스) → RRF 결합 → bge-reranker 재채점 → 게이트. 핵심파일 `app/services/nas_search/`(qdrant_search·query_embedder·reranker·bridge·hybrid), 라우터 `app/routers/nas_search.py`. **쿼리 임베딩·리랭커는 CPU(호스트)에서 동작 — 검색에 LLM 불필요.**
- **LLM**: 문서작업(docgen/form_filler)은 **Anthropic SDK 직접**(`app/services/llm/client.py`, `call_claude`, 모델 `claude-sonnet-4-6`/`claude-haiku-4-5`). 플레이그라운드 채팅은 **Tencent AIGC**(OpenAI호환, `text-aigc.vod-qcloud.com`). `call_claude`는 `temperature` 파라미터 지원(구조화 추출은 0).
- **문서 작업**: docgen(새 문서 생성) + form_filler(양식 채우기) 통합 surface. 공유코어 `app/services/documents/`(sources·nas_output·**extractor**). form_filler: analyzer(좌표앵커 `t{t}r{r}c{c}`/`p{i}` 발행)→mapper→renderer(앵커로 표셀 치환)→guardrails(grounding). docgen: generator(스켈레톤·judge루프)→markdown_blocks 렌더.

## 6. 핵심 노하우 / 함정 (시간 아끼는 것들)
- **dev 컨테이너에서 임베딩/리랭커 직접 호출 측정 금지** — 모델(8GB) 콜드로드가 매 `docker exec`마다 일어나 hang/좀비됨(D-state). **운영 컨테이너 `tk101-backend`는 lifespan 워밍업으로 모델이 떠있으니** 측정은 거기서. 라이브 진단: `docker exec tk101-backend python -c "from app.services.nas_search... embed_query/reranker.rerank/qdrant query_points"`.
- **워크트리 격리 에이전트는 npm build/pytest/alembic 불가**(node_modules·DB·dev컨테이너가 메인트리 전용). 워크트리에선 py_compile만, **통합은 메인트리에서 npm build+pytest+마이그 테스트 후 머지**.
- **병렬 에이전트가 BACKLOG.md를 같이 수정하면 머지 충돌** 발생. 해소: 브랜치에서 `git merge main` 후 `git checkout --theirs docs/BACKLOG.md`(main 기준 채택), 코드만 살림. 백로그는 통합자가 단독 관리 권장.
- **Qdrant 풀텍스트 인덱스 추가**(재임베딩 없이): `curl -X PUT localhost:6333/collections/docs_text/index -d '{"field_name":"text","field_schema":{"type":"text","tokenizer":"word","min_token_len":2,"lowercase":true}}'`. 160만 청크 ~1~2분 빌드, 무중단. (이미 text·path 인덱스 있음.)
- **검색 코드**: 임베딩(벡터)과 풀텍스트(텍스트)는 같은 청크에 공존, RRF로 병용(대체 아님). 키워드 arm 토글 `settings.nas_keyword_fulltext`.
- **antd v6**: `bodyStyle`(deprecated이나 동작) vs `styles={{body}}`. 코드베이스는 `bodyStyle` 혼용.
- **프론트 무거운 라이브러리**(mammoth 등)는 동적 import로 지연로딩 → 메인 번들 경량 유지.
- **운영 레포는 `tk101-dev`** (`tk101-dev-nas`는 낡은 사본, 쓰지 말 것).
