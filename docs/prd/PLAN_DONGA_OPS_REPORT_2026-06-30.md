# PLAN — 동아제약 운영보고서 자동 작성 (구글시트 데이터 → 기존 양식)

작성 2026-06-30 · 상태: **계획** · 대상 레포: **`tk101-dev`** (정상 PR→배포 흐름)

## 1. 목적 (오너 요구)
- 동아제약 운영 데이터는 **구글 시트**에서 관리 중.
- 그 데이터를 기반으로 **기존 운영보고서 양식에 맞게 자동 작성**하고 싶음.
- 데이터를 **자동으로 가져오면 좋지만 한계가 있어 보임** → 안 되면 **서버에 업로드 가능**.
- 오너가 **운영보고서 양식 데이터는 이미 확보**함. (BACKLOG의 "보고서 생성 — 양식 맞춤" 항목의 구체화.)

## 2. 현황 (조사 2026-06-30)
동아 워크스페이스는 **순수 프론트 스캐폴딩**(`DongaPharmaWorkspace.tsx`, 카드 3개). "보고서"·"시딩가이드"가 같은 `/forms/generate?wizard=1`로 중복 링크, "댓글분석" disabled, **동아 전용 백엔드/모델/데이터 연동 전무**.

**재사용할 핵심 인프라**
- **`form_filler`(양식 채우기)**: 빈 양식 템플릿 + 자료(data sources) → AI가 칸 자동 채움. 흐름: 양식 업로드 → form job → `POST /forms/jobs/:id/sources`(자료 등록) → `mapper`가 변수에 매핑 → `renderer` 출력. **운영보고서 자동작성의 정확한 안착점.**
- **양식 라이브러리**(`/forms/library`): 확보 양식을 템플릿으로 등록·재사용.
- 권한: `test_workspace` 모듈(admin 자동 + 부서 DB grant, 관리자 화면 체크).

## 3. 설계 — 2단계 (데이터 수급 방식으로 분리)

### Phase 1 — 수동 업로드 기반 (기존 인프라, 신규 인프라 0) ⭐먼저
```
확보한 운영보고서 양식 → 양식 라이브러리에 동아 템플릿으로 등록
구글시트 → (사용자가 xlsx/csv export) → form_filler 자료로 업로드
→ mapper가 양식 변수에 시트 데이터 매핑 → 자동 채움 → 검수 → 출력(xlsx/docx)
```
- **장점**: 오늘 있는 form_filler로 바로 동작, 실제 동아 데이터로 양식 채움 품질을 즉시 검증.
- **작업**: ①양식 템플릿 등록(데이터 작업) ②동아 워크스페이스 카드 "운영보고서 작성(양식 자동 채움)"을 form_filler 잡 생성으로 ready화 ③시트 export→자료 매핑이 운영보고서 구조(반복 행/월별 집계)에 맞는지 mapper 검증·보정.
- 전제: form_filler가 xlsx/csv 자료 소스를 받는지 확인(자료=form_data_sources 화이트리스트) → 표형 데이터 매핑 동작 점검.

### Phase 2 — 구글시트 자동 연동 (백엔드 신규)
```
구글 시트 API(서비스계정) → 시트 읽기 → 자동으로 form_filler 자료로 주입 → 자동작성
```
- **작업**: 백엔드에 Google Sheets API 통합(`gspread`/google-api-python-client + **서비스 계정 JSON**). 동아 담당자가 시트를 **서비스 계정 이메일에 공유**(또는 동아 워크스페이스에서 시트 URL 입력).
- **"한계"의 정체**: 시트 접근 인증(공유 권한)·시트 스키마 표준화. → Phase 1로 양식 채움이 검증된 뒤, 자동 pull은 "자료 수급 자동화"만 얹는 구조라 위험 격리됨.
- 동적 설계: 시트 URL/탭을 동아에 하드코딩하지 않고 워크스페이스 설정값으로.

## 4. 단계/난이도
| 단계 | 내용 | 난이도 |
|---|---|---|
| **P1a** | 확보 양식 → 양식 라이브러리 템플릿 등록 + 동아 카드 ready화(form_filler 연결) | 낮음(재사용) |
| **P1b** | 시트 export(xlsx/csv) → form_filler 자료 매핑이 운영보고서(표/월별)에 맞게 동작하도록 mapper/프롬프트 보정 | 중 |
| **P2** | 구글 시트 API 자동 연동(서비스계정, 시트 URL 입력) → 자료 자동 주입 | 중(신규 통합) |
| (선택) | 시딩가이드 전용 프리셋, 댓글분석(comment_analyzer+CSV/TXT) | 별도 |

## 5. 의사결정 / 오너 입력 필요
- **양식 파일 위치**: 확보한 운영보고서 양식 파일을 **서버 업로드** 또는 경로 전달 → 양식 라이브러리 등록.
- **시트 구조 샘플**: 구글시트 1개 샘플(또는 export xlsx) → 양식 변수↔시트 컬럼 매핑 설계.
- **자동 연동 시점**: Phase 1(수동) 먼저 검증 후 Phase 2(자동 pull) 결정 권장. 자동 연동 시 서비스 계정에 시트 공유 필요.

## 6. 다음 액션
**P1a + P1b** 착수 가능(기존 인프라). 시작 전 오너로부터 **①운영보고서 양식 파일 ②구글시트 샘플(또는 export)** 만 받으면 됨.
