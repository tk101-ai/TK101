# DESIGN — NAS 이미지/미디어 임베딩 (VLM 캡션 방식)

## 2. 접근 (오너 확정): VLM 캡션 → 텍스트 임베딩
멀티모달(CLIP) 대신 기존 텍스트 파이프라인 재사용. 이미지를 VLM으로 **한국어 캡션**화한 뒤, 그 캡션을 기존 Qwen3-Embedding-4B(2560d)로 임베딩 → `docs_image` 컬렉션에 upsert. 검색 시 텍스트 쿼리를 같은 인코더로 임베딩해 `docs_text` + `docs_image`를 **한 벡터공간에서** 동시 조회·병합.

장점: 인코더 1개로 양쪽 검색, 기존 청킹/멱등/적재/검색 코드 최대 재사용, 캡션이 사람이 읽는 출처로도 유용.

## 3. 데이터 모델
### 컬렉션 `docs_image` (신규 생성)
- 벡터: 2560d, cosine (docs_text와 동일 파라미터).
- point id: **`uuid5(NS, "{path}#img")`** (이미지 1장 = 1포인트, 멱등 재적재).
- payload:
  - `path`(canonical /mnt/nas/...), `dept`, `brand`, `year`(경로/분류에서), `size`, `mtime`
  - `modality="image_visual"` (config에 이미 정의된 값)
  - `caption`(임베딩 대상 텍스트), `caption_model`, `caption_ver`(프롬프트/모델 버전 — 재캡션 관리용)
  - `img_hash`(중복 제거·재캡션 판정용 파일 sha256[:16])
  - `thumb_path`(선택, 검색결과 썸네일용 다운스케일본 경로)

## 4. 캡션 파이프라인 (신규 모듈 `pipeline/caption_embed.py`)
```
이미지 파일
 → Pillow 로드·EXIF 회전보정·긴변 ~768px 다운스케일 → JPEG 재인코딩(토큰·전송 절약)
 → base64 data URL
 → VLM 캡션 프롬프트(아래) → 캡션 텍스트(한국어)
 → 기존 embed 파이프라인으로 2560d 임베딩
 → docs_image upsert (uuid5 멱등, 256개씩 배치 — 32MB 한도)
상태: caption_embed.done (path 집합) + img_hash 기록 → 재개·증분·중복제거 겸용
```
**캡션 프롬프트(초안)**: "이 이미지를 한국어로 검색 가능하게 묘사하라. 담긴 것(제품·인물·장면·브랜드), 이미지 내 텍스트(로고·카피·수치), 문서 유형(포스터·SNS카드·인포그래픽·사진), 색/분위기를 사실 위주로. 추측 금지." → 검색 recall에 직결되므로 P1에서 프롬프트 A/B.

## 5. 스코핑 필터 (대량 N 축소 — 비용 직결)
캡션 전 후보를 줄이는 결정적 필터(파일럿 후 임계 확정):
1. **크기 하한 ≥50KB** — 아이콘/썸네일 76k 제거(13%↓).
2. **폴더 우선순위** — 고객사 *최종 산출물/콘텐츠/게시물* 폴더 우선, `원본`·`raw`·`백업`·`소스` 후순위. (센서스 TSV로 폴더별 분포 산출 가능)
3. **중복 제거** — `img_hash` 동일 이미지 1장만(마케팅 자료 재사용률 높음 → 실효 큼).
4. (선택) 확장자 — heic/tiff 등 소량은 후순위.
→ 이 필터 적용 후 **"실제 캡션 대상 N"**을 산출해 P2 비용을 오너와 확정.

## 6. 검색 통합 (`tk101-dev` bridge)
- `search_relevant_chunks`에 `include_images` 옵션 → `docs_image`도 동일 쿼리벡터로 조회 → docs_text 결과와 **RRF 병합**(순위기반, 스케일 무관 — 다국어 병합과 동일 패턴 재사용 [[nas-search-tuning]]).
- 출처 표시: 캡션 + 경로 + 썸네일(있으면). modality 배지로 문서/이미지 구분.
- 채팅(rerank OFF)·docgen(rerank ON) 양쪽 적용 검토.

## 8. 리스크 / 미결
- **캡션 품질 = 검색 품질**. 나쁜 캡션은 recall 저하 → P1에서 실쿼리로 검증.
- 게이트웨이 vision 미검증 → Claude 직접이 안전 기본값.
- 배포가 Qdrant 재시작 → 캡션 적재도 재시도 래퍼+ckpt로 감싼다 [[embedding-jobs-ops]].
- 문서 내부 이미지(P3)는 디자인 PPT 가치 크나 추출 복잡 → 독립 파일(P2) 완료 후.
- 영상(mp4)은 희소 → 키프레임 캡션은 후순위(v2).
