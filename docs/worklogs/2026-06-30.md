## 3. 텐센트 AIGC PaaS 가이드 → AI 플레이그라운드 적용 (PR #161~162)
가이드 `https://doc.tencentpoc.com/justinkim/page`(7섹션 WebFetch). 상세=Text/Image/
Video/MPS/VOD, Music/3D=Coming Soon. 핵심·남은기회는 메모리 `tencent-aigc-guide.md`.

### ⭐ 핵심 발견
- **Claude는 `cd-` 접두사**로 게이트웨이 동작(`cd-sonnet-4.6`·`cd-opus-4.8`·`cd-haiku-4.5`). 과거 `claude-*` 400은 ID 오인이었음.
- 라이브 probe 200: cd-*(claude)·gemini-3.5-flash·3.1-flash-lite·deepseek-v4-pro/flash·kimi-k2.6·gk-4.3(Grok)·gpt-4o. **gpt-5.x=400(콘솔 미활성).**
- **ApiToken 계정당 최대 50개** — 재발급만 하고 미삭제해 풀이 차 LimitExceeded(발급 막힘) 발생. 수동 50→10 정리.
- 사용량/한도 API: `DescribeAigcUsageData`(AigcType별 Count/Usage)·`DescribeAigcQuotas`(QuotaType별) — 우리 자격증명으로 동작 확인.

### #2 즉시 보정 (PR #161)
- 생성 미디어 `StorageMode` 설정화(기본 **Permanent**) → i2i/i2v/v2v 재편집 베이스 URL 만료 해소. config `tencent_aigc_storage_mode`.
- 채팅 모델 카탈로그 확대(probe 200만): Claude provider 추가 + Gemini 3.5/3.1-lite·DeepSeek v4·Kimi k2.6·Grok gk-4.3·gpt-4o. `services/playground/__init__.py:PROVIDER_CHIPS`.
- ApiToken 50한도 자가복구: `token_manager._refresh`가 LimitExceeded 시 `_prune_tokens(keep5)`+재시도.

### #4 모니터링 대시보드 (PR #162)
- `services/playground/aigc_monitor.py`(DescribeAigcUsageData+Quotas 병렬, Text/Image/Video).
- `GET /api/playground/admin/aigc-monitor?days=`(require_admin).
- 프론트 `pages/playground/AigcMonitorSection.tsx`(UsagePage 상단, 타입별 카드+일별 스택막대+한도). 검증: 라이브 7일 텍스트 177회·103,095토큰·이미지 8.
