## 텐센트 AIGC 미디어 — 입력/조회 필드 (라이브 probe로 확정)

| 모드 | 생성 입력 필드 | 결과 조회 액션 |
|---|---|---|
| 텍스트→이미지 | VOD `CreateAigcImageTask` (임시저장) | `DescribeTaskDetail` |
| 텍스트→영상 (t2v) | VOD `CreateAigcVideoTask` (임시저장) | `DescribeTaskDetail` |
| 이미지→영상 (i2v) | MPS `CreateAigcVideoTask` + top-level **`ImageUrl`** | **`DescribeAigcVideoTask`** |
| 이미지 리터치 (i2i) | MPS `CreateAigcImageTask` + **`ImageInfos:[{ImageUrl}]`** | **`DescribeAigcImageTask`** |
| 영상 리터치 (v2v) | MPS `CreateAigcVideoTask` + **`VideoInfos:[{VideoUrl}]`** | **`DescribeAigcVideoTask`** |

- MPS는 결과를 **기본 COS 버킷(guangzhou)** 으로 자동 출력 → StoreCosParam 불필요(선택).
- VOD 영상 API는 입력 이미지/영상을 안 받음(ImageUrl/VideoInfos 거부) → i2v/v2v는 MPS 전용.

## 핵심 원인 규명 (시간 많이 쓴 것들)

1. **i2v "안 만들어짐"** = 결과조회 액션을 잘못 씀(`DescribeTaskDetail`). `DescribeAigcVideoTask`로 바꾸니 끝. (COS·권한 아님 — #136은 오판.)
2. **채팅 RAG가 엉뚱한 문서** = 인증/모델/게이트웨이 아님. 채팅이 사용자 **장황한 원문**을 그대로 벡터검색 → 무관문서 1건(0.084). 정제 "회사소개서"로 바꾸면 정상. (#141)
3. **채팅 느림** = CPU 리랭커가 18청크에 **~8.4s**(8s 타임아웃과 막상막하→가끔 0건). 채팅은 LLM이 다 읽으니 커버리지>정밀순위 → **리랭크 제거(0.29s)**. (#142)
4. **채팅 401 'Invalid API key'** = 토큰 **활성 지연**. `CreateAigcApiToken` 토큰은 발급 직후 401, **~10초 뒤 활성**. 재시작/55분갱신 직후 첫 대화가 그 윈도우에 걸림. (단일활성 아님 — 새 발급이 기존 무효화 X.) → 활성시각 추적+시작 워밍업+선발급으로 해결. (#143)

## ⚠️ 잔여/후속 (다음 세션 후보)

- **#1 v2v 충실도 낮음**: "애니 토끼 → 흰토끼 요청 → 실사 흰토끼". 텐센트 v2v(`VideoInfos`)는 **레퍼런스 기반 재생성**이라 원본 스타일이 약하게 반영(프롬프트 우세). 벤더 능력 한계.
  - 완화 후보: v2v 프롬프트에 "원본 영상의 **스타일·구성 유지**" 자동 prepend(미적용). #142로 원본 모델은 이미 유지.
- **#2 외부 영상 업로드 v2v**: 텐센트가 가져갈 URL **호스팅** 필요(생성영상 COS 서명 URL은 ~12h 만료, 재조회해도 서명 동일). 프레임 단위 "중간 수정"은 텐센트 미지원. (미구현)
- **RAG 정확도 잔여**: 리랭크 제거로 노이즈(아쿠아플라넷 바이두 등) 일부 유입, **TK101 자체 회사문서**(`/mnt/nas/COMPANY/`)가 고객사 소개서에 밀림.
  - 후보: 자체문서 **경로 가중치 부스트**, 정제 시 엔티티(회사명) 유지, 파일 dedup.
- **LLM 호출 타임아웃 부재**: 7일째 D상태 좀비(docgen `generate_document` 테스트)가 **타임아웃 없는 LLM 호출**에 매달려 있었음 → dev 컨테이너 재시작으로 제거. 같은 좀비 재발 막으려면 docgen/검증 LLM 호출부에 타임아웃 점검 권장.
