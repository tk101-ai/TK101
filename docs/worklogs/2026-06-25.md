## PR#117 — docgen NAS 검색 쿼리 정제
- 문제: docgen `topic`(자유지시문 "…나스에서 …찾아서 보고서 써줘")을 그대로 의미검색 쿼리로 써서 지시·형식 토큰이 임베딩을 흐림(전일 운영: "서버 증설"→신세계 홈페이지 오매칭 원인).
- 해결: `nas_search/query_refiner.refine_search_query` — 핵심 주제어만 **Haiku 1회(temp0, max80)** 추출해 **검색 입력에만** 사용(생성은 원본 topic 그대로 → 문서내용 불변). `collect_sources`(rag/both)에서 `asyncio.to_thread`로 호출.
- 안전: 비활성/<16자/예외/빈·과도(>200자)출력 → 원본 폴백(검색 안 멈춤). 설정 `docgen_query_refine_enabled`(기본ON)/`docgen_query_refine_model`. 단위테스트 8.
- 운영검증: "…신세계 서버 증설 관련 자료 찾아서 보고서 써줘" → "신세계 서버 증설".

## PR#119 — 슬라이드 레이아웃 아카이브 5종 (디자인퀄 1단계: 렌더만)
- 디자인퀄 = (A)렌더러 시각시스템(코드) + (B)AI 아트디렉팅(내용→레이아웃 선택). 1단계로 (A) 확보.

## 텐센트 게이트웨이로 docgen 생성 라우팅 — 검토 (PR 없음, 블로커 발견)
- 발단: 플레이그라운드 텐센트 통합 API를 디자인/PPT에 활용 가능? → "이미지 아니라 디자인 퀄리티"(위 #119/#120로 해결) → "텐센트 게이트웨이로 docgen 생성 라우팅 검토".
- 텐센트 능력: ①OpenAI호환 LLM채팅(`tencent_aigc_client.stream_chat`, 스트리밍 전용) ②AIGC 이미지/영상(`tencent_aigc_media`). 노출 모델(PROVIDER_CHIPS): gpt-5-chat·gpt-4.1·gemini2.5/3.x·glm5·kimi·minimax·deepseek. **Claude/Grok은 주석상 400("model not available")**, 단가표에도 Claude 없음.
- 오너: "텐센트도 Claude 가능한 걸로 안다, 비용·관리 고려하되 **품질 최우선**" → **라이브 probe 실측**:
  - secret_id/key/subapp(`1500063595`) 설정됨, ApiToken 발급 ✅.
  - 그러나 **텍스트 게이트웨이(`text-aigc.vod-qcloud.com`)가 gemini·gpt·claude 전부 401 "Invalid API key"** — 발급 토큰을 게이트웨이가 거부. Claude 가용 확인 불가(게이트웨이 자체가 안 받음).
- ⚠️ 부수 발견: 같은 `stream_chat`+토큰 경로라 **플레이그라운드 텍스트 채팅도 현재 401로 깨졌을 가능성**(운영 영향 가능, 미확인).
- 권고: 품질·캐싱비용(Anthropic 프롬프트 캐싱)·안정성 모두 현 Claude 직접 경로 우위 → **지금은 docgen Claude 직접 유지**. 텐센트 라우팅은 게이트웨이 정상화+Claude 가용 확인 후 가역적 설정게이트로 **A/B 실측** 후 결정.
