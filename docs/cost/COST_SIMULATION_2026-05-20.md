## 3. 사용자 페르소나별 월 비용

세 페르소나 정의:

| 페르소나 | 텍스트/일 | 이미지/일 | 영상/일 | 모델 가정 |
|----------|---------|---------|--------|---------|
| **L: 가벼운 사용자** | 5회 | 0 | 0 | `gemini-2.5-flash` |
| **M: 평균 사용자** | 20회 | 2장 | 0 | flash + Kling 2.1 |
| **H: 헤비 사용자** | 50회 | 5장 | 5초 영상 1편 | flash + Seedream + Kling 2.6 |

근무일 22일 가정.

| 페르소나 | 월 텍스트 | 월 이미지 | 월 영상 | **월 합계** | $10 한도 |
|----------|---------|---------|--------|-----------|---------|
| L | $0.34 | — | — | **$0.34** | ✅ 여유 |
| M | $1.36 | $0.62 | — | **$1.98** | ✅ 여유 |
| H | $3.41 | $4.18 | $4.62 | **$12.21** | ❌ **초과** |

> **결론**: 현재 $10/월 한도는 **L·M은 충분, H는 미달**.
> 모델을 비싼 것으로 바꾸면 M 도 빠르게 한도 도달.

| 항목 | 1인 월 | 20명 월 |

## 5. 비용 절감 레버 (효과 큰 순)

### Lever 1. 모델 라우팅 — **효과 ★★★★★**
같은 작업도 모델만 바꾸면 **20배 차이**.

| 비교 | 텍스트 1회 |
|------|----------|
| `deepseek-v3.2` | $0.00098 |
| `gpt-5-chat` | $0.01250 |
| 차이 | **12.8배** |

**전략**: 기본 모델을 `deepseek` 또는 `gemini-2.5-flash` 로 강제, 고급 모델은 admin 승인.

### Lever 2. 영상 모델·길이 제한 — **효과 ★★★★☆**
| 비교 | 10초 영상 |
|------|----------|
| Kling 2.6 | $0.42 |
| Kling 3.0 | $0.84 |
| Mingmou 1.0 | $3.00 |
| 차이 | **7배** |

**전략**: 기본 영상은 5초로 제한, 길이·고급 모델은 admin 권한.

### Lever 3. 캐시 활용 — **효과 ★★★☆☆**
cache read 단가는 input 의 **10분의 1**. 동일 시스템 프롬프트 재사용 패턴에서 효과 큼.

### Lever 4. 대화형 vs API 분기 — **효과 ★★★☆☆ (전략적)**
"대화형으로 충분한 작업은 ChatGPT/Gemini 구독으로" — 메모의 4번 항목.
API는 자동화·외부연동·결과 추출 등 **구독제로 안 되는 작업**에만 사용.

### Lever 5. 한도 단위 변경 — **효과 ★★☆☆☆**
현재 월 단위 → 일/주 단위로 잘게 쪼개면 폭주 방지에 효과적이나 사용자 경험은 저하.

## 6. 회의에서 결정 필요한 7가지

| # | 결정 항목 | 옵션 | 권장 |
|---|----------|------|------|
| 1 | 한도 단위 | 일 / 주 / 월 | **월** (현재 코드 유지) |
| 2 | 기본 한도 | $10 / $20 / $30 | **$15** (시나리오 A 평균 + 여유) |
| 3 | 한도 초과 시 | 즉시 차단 / 80% 알림 후 차단 / admin 승인 후 계속 | **80% 알림 + 100% 차단** |
| 4 | 기본 텍스트 모델 | flash / deepseek / glm-5 | **`gemini-2.5-flash`** (밸런스) |
| 5 | 고급 모델 접근 | 전체 허용 / admin 승인 / 그룹별 | **admin 승인** |
| 6 | 영상 기본 길이 | 5초 / 10초 / 사용자 자유 | **5초 기본, 10초 admin 승인** |
| 7 | 대화형 vs API 가이드 | 자유 / 가이드 문서 / 강제 분기 | **가이드 문서** (강제는 무리) |
