AI 에이전트 기반
사내 업무 자동화 시스템

멀티에이전트 아키텍처를 활용한 반복 업무 자동화 및 운영 효율화 방안

작성일 : 2026.04.15 버전 : v1.0

0프로젝트 개요

목적
AI 에이전트를 활용한 사내 업무 자동화 시스템 구축
범위
사내 전체 업무 취합 → 자동화 가능 업무 선별 → 순차 적용
핵심 원칙
한 번 구성하면 누구나 반복 사용 가능한 구조 (속인화 방지, 시스템화)
동시 처리 요건
여러 사람이 동시에 여러 업무를 요청해도 문제없이 처리 가능 — 멀티에이전트 / 에이전트 팀 / 병렬실행 방식 적용

사내 모든 업무를 취합하여 자동화 가능 여부를 구분하고, AI 에이전트를 통해 반복 업무를 시스템화합니다. 웹사이트 또는 AI 에이전트 인터페이스를 구성하여 담당자 한 명에 의존하지 않는 자립형 시스템을 구축하는 것이 최종 목표입니다.

1시스템 아키텍처

전체 시스템은 6개 계층으로 구성됩니다. 각 계층은 독립적으로 확장 가능하며, 사용자 요청부터 결과 전달까지의 전 과정을 커버합니다.

사용자
→
접속 화면
→
요청 관리
→
오케스트레이터
→
에이전트
→
결과 전달
1
사용자 접속 계층 (Frontend)
웹 대시보드(PC) · 모바일 반응형/PWA · 슬랙/카카오워크 챗봇 연동
2
요청 관리 계층 (API Gateway)
요청 접수 · 인증 · 작업 큐 관리 · 우선순위 배정
3
에이전트 오케스트레이터 (Control Tower)
작업 유형 판별 → 적합 에이전트 배정 → 병렬/순차 실행 결정
4
AI 에이전트 풀
업무별 전문 에이전트 (보고서/분석/검수/번역 등) · 동시 다수 실행 · 독립 작동
5
데이터·파일 저장소

6
결과 전달 계층
완료 알림 (슬랙/이메일/카카오) · 결과 파일 다운로드 링크

2작업 흐름 (End-to-End)

사용자 요청부터 결과물 수령까지 7단계로 구성됩니다. 완전자동화 업무는 STEP 4에서 바로 완료되며, 반자동화 업무는 중간에 사람 확인 단계가 삽입됩니다.

STEP 1. 업무 요청
사용자가 웹/모바일/챗봇에서 작업 유형 선택 + 파일 첨부 + 요구사항 입력
예: "경쟁사 분석 요청" + 게시물 텍스트 첨부
STEP 2. 요청 접수·분류
시스템이 작업 유형 자동 판별 (보고서/분석/검수/번역 등) + 긴급/일반 우선순위 배정
STEP 3. 에이전트 배정
오케스트레이터가 적합한 에이전트 선택. 복잡 작업은 멀티에이전트 병렬 실행
STEP 4. 작업 실행
에이전트가 자동 처리. 반자동화 작업은 중간에 사람 확인 단계 삽입
STEP 5. 품질 검증
결과물 자동 검수 — 맞춤법 / 가이드라인 준수 / 포맷 체크 / 에이전트 간 교차 검증
STEP 6. 결과 전달
완료 알림 발송 (슬랙/이메일/카카오) + 결과 파일 대시보드 게시 + 다운로드
STEP 7. 피드백 수집
사용자가 결과물 평가 (수정요청/승인) → 피드백이 에이전트 개선에 반영. 반복할수록 품질 향상

3초기 파이프라인 구성 순서

총 6단계(Phase)로 구성하며, Phase 1~5까지 약 7~11주 소요 예상입니다. Phase 6은 지속적 확장 단계입니다.

5멀티에이전트 처리 방식

여러 사람이 동시에 다양한 업무를 요청하는 환경에 대응하기 위해, 4가지 방식을 조합하여 적용합니다.

Q 작업 큐 기반 병렬처리

요청이 들어오면 큐에 적재 → 여러 에이전트 워커가 동시에 꺼내서 처리. 10명이 동시 요청해도 순차·병렬 혼합 처리 가능. 가장 안정적인 기본 방식.

T 에이전트 팀 구성

큰 작업(제안서 등)은 여러 에이전트가 분담. 구조설계 + 본문작성 + 검수 에이전트가 협업. 하나의 요청에 3~4개 에이전트 투입.

R 에이전트 라우팅

요청 내용을 분석하여 최적 에이전트에 자동 배정. "보고서 써줘" → 보고서 에이전트로 자동 연결. 사용자는 세부 선택 불필요.

C 동시성 제어

동시 실행 수 제한 + API 일일 비용 상한 설정 + 우선순위 큐. 비용 폭증과 시스템 과부하를 사전에 방지.
