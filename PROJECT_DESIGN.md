# TK101 사내 AI 자동화 플랫폼 - 프로젝트 설계 문서

## 프로젝트 개요

TK101 Global Korea Inc. 사내 전용 AI 자동화 플랫폼 구축 프로젝트.
40명 규모 회사의 다양한 업무를 AI로 자동화하고, 중앙화된 데이터 관리와 권한 제어를 제공하는 통합 플랫폼.

---

## 핵심 원칙

- **혼자 유지보수 가능한 구조** : 담당자 1인(TK101)이 Claude Code를 활용해 구축 및 유지
- **작게 시작, 점진적 확장** : 첫 모듈부터 확장 가능한 뼈대 위에서 구축
- **Claude Code 친화적 설계** : 새 세션에서도 이 문서만 보면 전체 구조 파악 가능

---

## 플랫폼 구조 (3 레이어)

### 1. 인터페이스 레이어 (사용자 접점)

| 채널 | 용도 | 우선순위 |
|------|------|----------|
| 웹 플랫폼 | 메인 채널. 작업 요청, 결과 확인, 대시보드, 백오피스 | 1순위 |
| 카카오톡 | 서브 채널. 간단한 업무 요청 및 결과 수신 | 2순위 |

**웹 플랫폼 구성:**
- 일반 직원용: 작업 요청 (자연어 + 자주 쓰는 작업 버튼), 결과 확인, 대시보드
- 백오피스(관리자용): 사용자/부서/권한 관리, 시스템 설정, 작업 히스토리

### 2. AI 처리 레이어 (중앙 허브)

- **현재**: Claude API 중심
- **추후 확장**: OpenAI API, Gemini API 멀티 AI 구조
- **설계 원칙**: AI 엔진을 쉽게 교체/추가할 수 있는 추상화 레이어 필요
- **자율도 레벨**:
  - 완전 자율: 데이터 기반 처리형 (규칙 명확, 리스크 낮음)
  - 반자율: AI 초안 생성 → 사람 검토 후 확정
  - 보조형: AI가 자료 수집/정리, 사람이 최종 작성

### 3. 데이터 레이어 (저장 및 관리)

- **서버**: Tencent Cloud CVM (신규 인스턴스)
- **스토리지**: 사내 NAS 연동 검토 중 (외부 접근 가능 여부 확인 필요)
- **DB**: 미정 (기술 스택 선택 시 함께 결정)
- **권한 기반 접근 제어**: 직원별 열람 가능 데이터 범위 제한

## 개발 단계 계획

### 1단계 - 뼈대 구축 (최우선)
- [ ] Tencent CVM 서버 세팅
- [ ] 기술 스택 결정 (Python 권장, 레퍼런스 풍부, AI 라이브러리 풍부)
- [ ] Claude API 연결
- [ ] 기본 웹 인터페이스 (로그인 + 기본 화면)
- [ ] DB 구축 (사용자, 부서, 권한 테이블)
- [ ] 사용자/권한 관리 백오피스 기본 구현

### 2단계 - 첫 번째 모듈: 회계 자동화 (빠른 결과물)
- [ ] 은행 거래내역 파일 업로드 및 파싱 (KB국민, 기업, 농협, 신한, 우리, 하나)
- [ ] 중복 제거 후 신규 내역만 자동 추가
- [ ] 자사 계좌간 이체 자동 감지 및 표시/메모 ← **이사님 요청, 최우선**
- [ ] 세금계산서(매출/매입) 내역과 은행 내역 자동 매칭
- [ ] 결과물 이중 출력: 웹 대시보드 + 엑셀 리포트

### 3단계 - 플랫폼 확장
- [ ] 카카오톡 채널 연동
- [ ] NAS 연동 (외부 접근 확인 후)
- [ ] 추가 업무 모듈 순차 탑재 (마케팅, 디자인, 신사업팀 등)
- [ ] 멀티 AI 구조 (OpenAI, Gemini 추가)

## 회계 자동화 모듈 상세

### 현재 파일 구조
```
거래내역 계좌/
  ├── KB국민/
  ├── 기업/
  ├── 농협/
  ├── 신한/
  ├── 우리/
  └── 하나/
      └── TK101(은행 계좌번호)_분기.xlsx

세금계산서 내역/
  ├── 매입세금계산서목록_2026.xlsx
  ├── 매출세금계산서목록_2026.xlsx
  ├── 세금계산서_기입내역_검증용.xlsx
  └── 세금계산서_매칭결과_검증용.xlsx

거래내역 특이사항/
  └── 계좌간이체_기입내역_검증용.xlsx
```

### 자동화 흐름
1. 실무자가 은행 파일 다운로드 후 해당 폴더에 저장
2. 시스템이 새 파일 감지
3. 기존 데이터와 비교 → 중복 제거 → 신규 내역만 추가
4. 자사 계좌번호 목록과 대조 → 계좌간 이체 자동 표시
5. 세금계산서 내역과 매칭
6. 웹 대시보드 업데이트 + 엑셀 리포트 생성

### 사용 은행 목록
- KB국민은행
- 기업은행
- 농협은행
- 신한은행
- 우리은행
- 하나은행

## GitHub 계정 구조

- **개인 계정 사용 금지**: 퇴사 시 접근 권한 소멸 리스크
- **회사 GitHub Organization 사용**: 처음부터 회사 명의 Organization 생성 후 레포 관리
- **소유권 이전 용이**: 퇴사 시 Organization 소유권만 이전하면 CI/CD 포함 전체 이전 가능
- **API 키 및 서버 접근 키**: 개인 계정이 아닌 회사 명의로 관리 (GitHub Secrets에 등록)
- **인수인계 체크리스트** (퇴사 시):
  - GitHub Organization 소유권 이전
  - Tencent CVM 접근 키 이전
  - API 키 (Claude, OpenAI, Gemini) 재발급 및 이전
  - 카카오톡 채널 관리자 이전

## CI/CD 구성

- **도구**: GitHub Actions
- **목적**: 코드 변경 시 자동으로 Tencent CVM에 배포
- **기본 흐름**: 로컬/Claude Code에서 코드 수정 → GitHub push → GitHub Actions 자동 실행 → 서버 배포
- **장점**: 서버에 직접 접속해서 수동 배포할 필요 없음, 배포 이력 관리 가능
- **1단계에서 함께 구성**: 서버 세팅 시 GitHub 연동 및 Actions 워크플로우 파일 함께 설정

## 참고 사항

- 은행 사이트 직접 접속 자동화는 보안상 진행하지 않음
- 파일 다운로드는 실무자가 수동으로, 이후 처리만 자동화
- 외부 멘토 존재하나 코드 리뷰 의존도 낮게 유지
- 모든 데이터는 사내에서만 처리 (외부 유출 최소화)
