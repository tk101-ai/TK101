# AI 자동화 인프라 구축 메모

## 기본 정보
- 직원 수: 약 40명
- 직원 기술 수준: 비개발자 (IT 종사자 거의 없음)
- 기존 메신저: 잔디(JANDI)
- 대표 방향: Claude 에이전트 기반 병렬 자동화
- 담당자 방향: 인프라 구축 우선

---

## 아키텍처 개요

```
직원 (40명)
  │
  ├─ 웹 채팅 (메인 인터페이스)
  └─ 잔디 or 슬랙 (메신저 연동)
        │
        ▼
  게이트웨이 서버
  ├─ 인증 (누가 보냈는지)
  ├─ 비용관리 (부서별 한도)
  └─ 요청 큐 (동시요청 처리)
        │
        ▼
  Claude API
  ├─ Haiku  (간단한 질문) ← 80% 사용 목표
  ├─ Sonnet (일반 업무)
  └─ Opus   (복잡한 분석, 필요시만)
```

## 비용 추정

### Claude API 비용 (월 기준)
- 40명 × 하루 5회 × 22일 = 4,400건/월
- 요청당 평균: 입력 1,000 + 출력 2,000 토큰
- Haiku 80% + Sonnet 20% 기준: 약 $64/월 ≈ 약 9만원
- 사용량 증가 고려 시: 15~30만원 예상

### 서버 비용 (월 기준)
- 클라우드 서버: 월 1~5만원
- 도메인/SSL: 연 1~2만원

## 인터페이스 비교: 웹채팅+잔디 vs 슬랙 전환

### 비용 비교

| 항목 | 웹채팅 + 잔디 | 슬랙 전환 |
|------|-------------|----------|
| 메신저 비용 | 잔디 기존 요금 그대로 | 슬랙 Pro $7.25/인 × 40명 ≈ 월 42만원 |
| 서버 비용 | 웹 채팅 서버 월 3~5만원 | 거의 없음 (슬랙이 호스팅) |
| Claude API | 월 9~30만원 | 월 9~30만원 |
| **월 추가 비용** | **3~5만원** | **약 42만원** |
| **AI 포함 월 총 예상** | **약 12~35만원 + 잔디 기존 요금** | **약 51~72만원** |

### 기능 비교

| 기능 | 웹채팅 + 잔디 | 슬랙 전환 |
|------|-------------|----------|
| AI 대화 품질 | 자유로운 대화형 UI | 슬랙 채팅 내 봇 대화 |
| 파일 첨부/공유 | 웹에서 자유롭게 | 슬랙 내 자연스럽게 |
| 대화 히스토리 | 직접 구현 필요 | 슬랙이 자동 보관 |
| 인터랙티브 UI | 직접 구현 (자유도 높음) | 버튼, 모달, 드롭다운 기본 제공 |
| 슬래시 커맨드 | 없음 (웹 UI로 대체) | /ask /요약 등 가능 |
| 업무 연동 확장 | 별도 개발 필요 | 슬랙 앱 생태계 (구글, 노션 등) |
| 알림/공유 | 잔디 Webhook으로 전달 | 채널에 바로 공유 |
| 봇 개발 난이도 | 웹 프론트엔드 + 백엔드 | 슬랙 SDK로 비교적 간단 |

### 도입 난이도 비교

| 항목 | 웹채팅 + 잔디 | 슬랙 전환 |
|------|-------------|----------|
| 직원 학습 비용 | 웹 UI 사용법만 (낮음) | 슬랙 자체를 새로 배워야 함 (높음) |
| 기존 업무 영향 | 잔디 그대로, 웹만 추가 | 메신저 자체를 전환 |
| 개발 공수 | 웹 UI 구축 (2~4주) | 슬랙 봇 구축 (1~2주) |
| 유지보수 | 웹 서버 + 봇 서버 관리 | 슬랙 봇만 관리 |

### 리스크 비교

| | 웹채팅 + 잔디 | 슬랙 전환 |
|---|-------------|----------|
| 최대 리스크 | 직원들이 웹에 안 들어옴 | 메신저 전환 저항 (40명 업무 중단) |
| 확장성 | 기능 추가 시 직접 개발 | 슬랙 생태계로 쉽게 확장 |
| 비용 | 낮음 (서버비만) | 슬랙 요금 고정 지출 |
| 기술 | 웹 UI 개발/유지 역량 필요 | 슬랙 API 의존 |

### 요약
```
웹채팅 + 잔디가 유리한 경우:
  → 비용을 AI API에 집중하고 싶을 때
  → 잔디 전환이 현실적으로 어려울 때
  → AI 인터페이스를 자유롭게 커스터마이징하고 싶을 때

슬랙 전환이 유리한 경우:
  → 장기적으로 업무 도구 통합을 원할 때
  → 개발/유지보수 인력이 부족할 때 (생태계 활용)
  → AI 외에도 슬랙 앱 연동을 활용하고 싶을 때
```

## 잔디 API 현황
```
지원:
  ✅ Incoming Webhook (외부 → 잔디 메시지)
  ✅ Outgoing Webhook (키워드 감지 → 외부 서버)

미지원:
  ❌ Bot User API
  ❌ 인터랙티브 UI (버튼, 모달)
  ❌ 실시간 WebSocket
  ❌ Slash Command
```

## 구축 로드맵

```
Step 1: API 키 발급 + 파일럿 봇 (1~2일)
        → 5명 파일럿 시작

Step 2: 비용 추적 대시보드 (1주)
        → 사용량 모니터링

Step 3: 프롬프트 템플릿 세팅 (업무 리스트 회의 후)
        → 업무별 맞춤 프롬프트

Step 4: 전사 오픈 (파일럿 검증 후)
        → 교육 + 가이드 배포
```
